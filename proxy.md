# Протокольный прокси

Используется в случае если передать файл или совершить звонок напрямую не получается.

## `MRIM_CS_PROXY` (0x1044) C <-> S

Просьба организовать прокси. Двухнаправленный

Команду может отправить сервер, если другой клиент запросил прокси.

| Тип данных | Примерное содержание | Значение          |
| ---------- | -------------------- | ----------------- |
| LPS        | `example@mail.ru`    | От кого или куда  |
| UL         | N/A                  | Уникальный идентификатор сессии   |
| UL         | N/A                  | Тип прокси   |
| LPS        | N/A       | Данные (в случае с передачей файлов - их список) |
| LPS        | N/A       | Айпи (со стороны клиента пустые) |
| UL x 4     | N/A                  | Уникальный идентификатор сессии для прокси   |
| LPS        | N/A       | (при передаче файлов - их список в юникоде) |

Типы прокси:

- 0x1 - звонок
- 0x2 - передача файлов
- 0x3 - звонок 2 (?)

## `MRIM_CS_PROXY_ACK` (0x1045) S -> C

Положительный ответ на просьбу организовать прокси

Команду может отправить сервер, если другой клиент запросил прокси.

| Тип данных | Примерное содержание | Значение          |
| ---------- | -------------------- | ----------------- |
| UL         | N/A                  | Статус   |
| LPS        | `example@mail.ru`    | От кого или куда  |
| UL         | N/A                  | Уникальный идентификатор сессии   |
| UL         | N/A                  | Тип прокси   |
| LPS        | N/A       | Данные (в случае с передачей файлов - их список) |
| LPS        | N/A       | Айпи в формате IP:PORT; |
| UL x 4     | N/A                  | Уникальный идентификатор сессии для прокси   |
| LPS        | N/A       | (при передаче файлов - их список в юникоде) |


Возможные статусы:

- 0x0 - отклонено
- 0x1 - принято
- 0x2 - ошибка
- 0x3 - несовместимые версии
- 0x4 - нет свободных ресурсов

## `MRIM_CS_PROXY_HELLO` (0x1046) C -> S

Отправляется клиентом после подключение к прокси, выданному сервером, НА ПРОКСИ.

**Важный нюанс** - для прокси версия протокола в заголовке своя. В MRA 5.x и 6.x это 1.8.

| Тип данных | Примерное содержание | Значение          |
| ---------- | -------------------- | ----------------- |
| UL x 4     | N/A                  | Уникальный идентификатор сессии для прокси   |

## `MRIM_CS_PROXY_HELLO_ACK` (0x1047) S -> C

Отправляется проксирующим сервером, когда подключился второй клиент и всё шик. Данных не имеет. После этой команды оба клиента соединяются pipe'ом
